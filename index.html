SELF-CUSTODIAL

Your BTC will remain in your wallet until the transaction is complete, with no additional risk, and will only leave when you want to exchange it.

Lightning Payment

Lightning Network enables instant transactions. Atomic swaps based on HTLC technology guarantee completion, but only when both parties confirm payment. If both parties don’t confirm then the transaction is cancelled.

How it works

HTLC is the foundation of the Lightning Network and an embedded programming script in both the Lightning Network and BTC mainnet. BCDEx's atomic trading protocol, based on HTLC and smart contracts, enables self-custodial, decentralized liquidity provision without a fund pool or oracle. Anyone can trade BTC on BCDEx or become a market maker without having to deposit BTC in an insecure exchange or DeFi protocol.

Async Atom Swap

Each BTC or Lightning Network transaction can be set with an unlock password (required to access the funds). Similarly, smart contracts can be used on Ethereum-compatible blockchains. Example: Alice wants to exchange BTC with Bob.

  • They agree upon a certain amount.

  • Alice sends a transaction with a password attached.

  • PAlice then provides the hash of the password to Bob.

  • Bob uses the hash as a condition for unlocking the transaction he sends to Alice.

  • When Alice unlocks Bob's transaction, he receives the password and can unlock Alice's transaction.

Security

Transfer transactions have time locks, and if the counterparty does not pay, your transfer will be returned after a set time. Your funds remain in your wallet until you authorise the exchange. The procedure is based on the basic scripts of BTC and the Lightning Network, which have been running for years and have proven robust in practice.
