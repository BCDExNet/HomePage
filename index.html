How it works

HTLC is the foundation of the Lightning Network and an embedded programming script in both the Lightning Network and BTC mainnet. BCDEx's atomic trading protocol, which is based on HTLC and smart contracts, enables self-custody, decentralized liquidity provision without a fund pool or Oracle. Anyone can trade BTC on BCDEx or become a market maker without having to deposit BTC in an insecure exchange or Defi protocol.

Async Atom Swap

Each BTC or Lightning Network transaction can be set with an unlock password which is required to access the funds. Similarly, smart contracts can be used on the Ethereum blockchain. When Alice wants to exchange BTC with Bob, they agree upon a certain amount and Alice sends a transaction with a password attached. Alice then provides the hash of the password to Bob, who also uses the hash as a condition for unlocking the transaction he sends to Alice. When Alice unlocks Bob's transaction, he receives the password and can then unlock Alice's transaction.

Security

Transfer transactions have time locks, and if the counterparty does not pay, your transfer will be returned after expiration. Your funds remain in your wallet unless you want to exchange. The script used is based on the basic scripts of BTC and the Lightning Network, which have been running for years and have been proven in practice.
